---
title: リリースチェックリスト
description: リリース前後に確認すべき共通項目をまとめたチェックリストです。
---

## リリース準備

- [ ] 変更内容がプロダクトバックログやチケットと整合している
- [ ] 自動テスト（unit/e2e/visual）が成功し、必要な手動確認も完了している
- [ ] ドキュメント・リリースノート・サポート情報が更新済み
- [ ] Feature Flag / 環境変数の設定が本番環境に適用されている

## デプロイ前チェック

- [ ] `bun run build` が成功し、サイズ増加や警告をレビュー済み
- [ ] `bun run cf:dry-run` で Cloudflare へのデプロイスクリプトが問題なく実行できる
- [ ] ログ・アラート設定の変更点が反映されている
- [ ] ロールバック手順と担当者が明確になっている

## リリース後フォロー

- [ ] モニタリングダッシュボードで主要指標をチェックし、異常がないか確認
- [ ] サポートチャネルへリリース完了を通知し、問い合わせ対応体制を共有
- [ ] ふりかえりで得られた学びを関連ドキュメントに追記

## エスカレーションフロー

1. 障害や重大な不具合を検知した場合、オンコール担当に即時連絡
2. 影響範囲と暫定対応を 30 分以内に定義し、ステークホルダーに共有
3. ロールバックまたは Hotfix を実施し、事後にポストモーテムを作成

このチェックリストは随時更新し、運用ルールの変更があれば履歴を残してください。
