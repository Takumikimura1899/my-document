## このカテゴリの目的

このカテゴリは、チームに参加したメンバーが最短でキャッチアップできるように「まず読むべき」情報を集約しています。以下の観点を中心に構成しており、詳細は各記事で掘り下げています。

- 開発環境の整え方と、初回セットアップ時にハマりやすいポイント
- プロジェクト全体で共有しているコーディング標準とレビューの観点
- リリース前に必ず確認するチェックリストと、緊急時の対応フロー

## 推奨する読み進め方

1. [開発環境セットアップ](/docs/essentials/environment/)でローカル環境を構築する
2. [コーディング標準](/docs/essentials/coding-standards/)でチームのコード規約を確認する
3. [SOLID 原則](/docs/essentials/solid/)で設計時の共通前提を理解する
4. [リリースチェックリスト](/docs/essentials/release-checklist/)でリリース手順と責務範囲を把握する

## 他カテゴリとの関係

- チームの価値観やコミュニケーションルールは[はじめに](/docs/intro/)カテゴリで詳しく解説しています。
- フロントエンド・バックエンド特有のガイドは、それぞれのカテゴリから参照してください。
- テストや運用に関する詳細なノウハウは [Test](/docs/test/) や [Other](/docs/other/) に整理しています。

ここに記載の内容は定期的に見直し、直近のナレッジや改善内容を反映させてください。
