## シナリオ設計

- ユーザーが重視するコアジャーニーを選定する
- クリティカルパスに加えて代替フローやエラーパターンも検証する
- データ依存を減らし、テスト間の独立性を保つ

## 実行環境

- 本番に近い設定で実行し、外部サービスはスタブを活用する
- 並列実行による時間短縮と競合のバランスを取る

## 継続的改善

- テストの失敗理由を定期的に分析し、安定性を高める
- UX の変化に合わせてシナリオを更新する
- テスト結果をダッシュボード化し、チームで共有する
