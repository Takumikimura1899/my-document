## テストの目的

- ユーザーに価値を届けるスピードを保ちながらリスクを抑える
- 回帰を早期に検知し、デプロイ後の不具合を最小限にする
- 品質指標を定義し、継続的に改善する

## レイヤー構成

- Unit: ロジックの最小単位を高速に検証
- Integration: コンポーネント間の協調動作を確認
- E2E: 実ユーザーのシナリオを通じた振る舞い検証
