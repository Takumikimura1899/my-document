## 対象範囲

- ブラウザ上で動作するアプリケーション全般（SPA、MPA、PWA など）
- サーバーサイドレンダリング（SSR）や静的サイト生成（SSG）を含む場合もある
- UI/UX デザインの実装と連携
- UI コンポーネント（React など）とデザインシステムの連携
- パフォーマンス最適化、アクセシビリティ、国際化対応

## 開発の基本姿勢

- コンポーネントは再利用性を意識して設計し、Storybook 等で可視化する
- デザインとの同期を保つため、デザインツールとドキュメントを往復する仕組みを整える
- ビルド時の警告や lint エラーは常に解消する

## 技術選定基準

- React, TypeScriptをデファクトスタンダードとする
- 要件に応じてAstroやSvelteなどの採用も検討する

### 指針

- フルスタックSSR: Next.js / React Router / Tanstack Start
- SSG, MPA: Astro
- 軽量SPA: Vite + React / Svelte
