---

## 設計原則

- 単一責任: コンポーネントは 1 つの明確な目的に絞る
- 明示的な API: props の型定義とドキュメントを整備する
- パフォーマンス: 不要な再レンダリングを防ぐためメモ化や分割を検討する

## 実装ガイド

1. UI ステートとビジネスロジックを切り離す
2. Storybook / Playground でバリエーションを可視化する
3. アクセシビリティ属性（`aria-*`など）を確認する

## レビュー観点

- props の命名は意図が伝わるか
- コンポーネントの責務が過剰になっていないか
- テスト（スナップショット・相互作用）が用意されているか
