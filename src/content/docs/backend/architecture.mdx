## コンポーネント構成

- プレゼンテーション層: API Gateway / Edge Functions
- アプリケーション層: ドメインサービス、ユースケース
- データ層: 永続化ストア、キャッシュ、外部サービス連携

## 設計指針

- 責務を明確に分割し、変更の影響範囲を小さく保つ
- 非同期処理やバッチ処理はリトライ戦略を定義する
- SLA / SLO を意識した設計を行い、ボトルネックを定期的にレビューする

## ドキュメント

- 図面やシーケンス図を併用し、俯瞰できる形で共有する
- 変更があれば ADR（Architecture Decision Record）に記録する
