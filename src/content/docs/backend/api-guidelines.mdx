title: API ガイドライン
description: 安定した API を設計・運用するための指針です。
---

## 設計原則

- 一貫したリソース設計（REST/GraphQL いずれでも命名と責務を明確に）
- バージョニング方針を定義し、互換性の破壊は計画的に行う
- エラーレスポンスは機械判読可能な構造にする

## セキュリティ

- 認証・認可を標準化し、役割ごとの権限を明確に管理する
- レートリミットやスロットリングを適用し、乱用を防ぐ
- データ検証とサニタイズを共通レイヤーで実施する

## 運用

- SLA/SLO に基づくメトリクスを計測し、アラートを設定する
- API 変更時はクライアントへの告知と移行期間を確保する
- ドキュメント（OpenAPI/GraphQL Schema）を自動生成し、常に最新に保つ
