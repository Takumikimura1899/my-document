## チャネルの使い分け

- **Slack**: 日常の連絡、即時性の高い質問、進行中タスクの共有
- **Issue / Pull Request**: 意思決定の履歴や技術的な議論の記録
- **ドキュメント**: 方針・手順など長期的に参照する情報

## レビューとフィードバック

- 目的を先に述べ、どこに着目してほしいかを明示する
- コメントは具体的な改善提案と背景をセットで伝える
- レビューの完了条件（テスト結果、確認内容）を明文化する

## オンラインミーティング

- アジェンダと期待されるアウトカムを事前に共有する
- 会議後は要点と決定事項、アクションアイテムをまとめて公開する
- 参加できなかったメンバーもキャッチアップできるように情報を残す

## ナレッジシェア

- 気づきや学びは週次で共有する場を設ける
- ドキュメント更新時は変更点をチームにアナウンスする
- ドキュメントにコメントや質問を残し、非同期な議論を活用する
